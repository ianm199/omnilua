One snippet. Five Luas. In your browser.

omniLua is one pure-Rust runtime for Lua 5.1, 5.2, 5.3, 5.4, and 5.5 — as a standalone interpreter, embedded in Rust, or in the browser. No C dependency, no unsafe FFI.

If your Rust program — or your game — ships a wasm build, a C-backed Lua binding can't follow it. omniLua is pure Rust: the same scripting runtime compiles natively and to wasm32-unknown-unknown, with no Emscripten and no toolchain gymnastics.

snippet.lua Cmd/Ctrl + Enter to run
Lua 5.4 live 
Output appears here. Press Run or Cmd/Ctrl+Enter.

What's live vs reference: the shipped .wasm bundle currently runs the Lua 5.4 backend only, so the 5.4 column executes your edited code live. The 5.1/5.2/5.3/5.5 columns show verified reference output for the default snippet — captured from the official PUC-Rio reference binaries — and are labelled ref. Runtime per-instance version selection across all five backends is the live frontier (see the multi-version API spec); when the multi-backend .wasm ships, every column runs live with no page change.

Get omniLua

Every Lua, everywhere — pure-Rust Lua 5.1–5.5, suite-passing, LuaRocks-compatible, wasm-ready.

Rust CLI
$ cargo install omnilua-cli
Embed in Rust
# omnilua = "0.1"
Browser / Node
$ npm install omnilua
GitHub crates.io npm Lua

~1.3× geomean of reference C wall time — competitive, not faster, and not LuaJIT. If you need LuaJIT speed or a decades-mature binding, use mlua.
